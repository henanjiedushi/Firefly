Obsidian + GitSync 手机端博客方案：Hugo + GitHub + Cloudflare Pages

很多人搭建完 Hugo 博客后，会遇到一个实际问题：

> 电脑端写作和部署没问题，但手机端写博客体验非常割裂。

传统流程通常是：
本地 Hugo → Git 提交 → GitHub → Cloudflare Pages 自动部署

但如果想在手机上随时记录、发布文章，就会发现：手机 Git 工具体验差，Hugo 本地编译困难，Markdown 编辑器与 Git 不联动，图片管理混乱，发布流程繁琐

后来我逐渐整理出了一套比较稳定的方案：

Obsidian
（Markdown 写作）
        ↓
GitSync
（自动 Git 提交同步）
        ↓
GitHub Repository
（博客源码仓库）
        ↓
Cloudflare Pages
（自动拉取构建 Hugo）
        ↓
公网博客站点

这套方案最大的优势是：
手机端体验接近普通笔记软件
不需要手机本地部署 Hugo
无需手动 push
支持多设备同步
完全基于 Git 工作流

本文会完整介绍这一套方案的架构、原理以及实际使用体验。

---

## 一、整体架构

先看整个博客工作流：

Obsidian
（Markdown 写作）
        ↓
GitSync
（自动 Git 提交同步）
        ↓
GitHub Repository
（博客源码仓库）
        ↓
Cloudflare Pages
（自动拉取构建 Hugo）
        ↓
公网博客站点

整个流程中：

|        组件        |     作用      |
| :--------------: | :---------: |
|     Obsidian     | Markdown 写作 |
|     GitSync      | 手机 Git 自动同步 |
|      GitHub      | 存储 Hugo 源码  |
| Cloudflare Pages |   自动部署博客    |
|       Hugo       |   静态博客生成器   |



---

## 二、为什么选择 Obsidian
### 1. Markdown 原生支持

Hugo 本质上就是 Markdown 驱动。

而 Obsidian：，支持实时 Markdown 编辑，支持双链，支持附件管理，支持代码块，支持 YAML Front Matter和 Hugo 的兼容性非常高。

例如：
---
title: "测试文章"
date: 2026-05-25
tags:
  - Hugo
  - Obsidian
---

# Hello Hugo

Obsidian 可以直接编辑。无需转换格式。
---

### 2. 文件系统开放

很多笔记软件的问题在于：

> 数据不是“真实文件”。

而 Obsidian 的 Vault 本质就是：

/content/posts/article.md

真实存在于文件系统中的 Markdown 文件。

这意味着：Git 可直接管理，Hugo 可直接读取，GitHub 可直接同步
这是它特别适合博客写作的核心原因。

---

## 三、为什么不用手机本地部署 Hugo

很多人最开始会尝试：
手机：
Termux + Hugo + Git

理论可行，但实际体验会出现很多问题：

| 问题        | 原因           |
| --------- | ------------ |
| Hugo 安装复杂 | ARM 环境兼容问题   |
| 图片路径混乱    | Android 文件权限 |
| Git 操作麻烦  | 手机终端体验差      |
| 本地预览困难    | 缺乏完整开发环境     |
| 容易误操作     | 手机 shell 容错低 |

因此更合理的方案是：

> 手机只负责写作与同步。

真正的 Hugo 构建交给：
GitHub Actions
或 Cloudflare Pages

这也是现代 Jamstack 的典型设计思路。
---

## 四、GitSync 的核心价值

GitSync 是整个手机博客方案里最关键的一环。
它解决了：

> “手机端 Git 太难用” 的问题。

---

GitSync 的作用
它本质上是：Android Git 自动同步工具
可以实现：自动 pull，自动 commit，自动 push，定时同步，后台同步

因此：
你在 Obsidian 写完文章后：

保存
↓
GitSync 自动同步
↓
GitHub 更新
↓
Cloudflare 自动部署
↓
博客上线

整个过程几乎不需要人工干预。

## 六、为什么选择 Cloudflare Pages

相比 VPS 自部署：Cloudflare Pages 的优势非常明显。

---

### 1. 自动部署

只要 GitHub 有 commit：
Cloudflare Pages 自动重新构建
完全不需要：SSH,FTP,rsync

---

### 2. 免费 CDN

Cloudflare 本身就是全球 CDN。
静态博客非常适合：缓存，边缘分发，Pages 托管

---

### 3. Hugo 官方支持较好

Pages 对 Hugo 支持比较完善。
通常只需要：构建命令，hugo，输出目录，public即可。

## 八、目前存在的问题

这套方案并非完全没有缺点。

---

### 1. 图片管理仍然偏麻烦

Obsidian 手机端：图片压缩能力弱，自动重命名一般，批量管理困难。因此建议上传前先压缩 WebP，统一放 static/images。文件名规范化

例如：
2026-05-25-gitsync-workflow.webp

---

### 2. Git 冲突问题

如果手机正在编辑，电脑也在编辑可能出现：merge conflict
因此建议写作前先同步，不同时修改同一篇文章

---

### 3. Obsidian 大仓库可能卡顿

尤其图片过多，插件过多，文件过多，手机端会明显变卡。
因此建议不要把整个资源库塞进 Vault，控制插件数量，图片尽量外链或压缩

---

## 九、适合什么人

这套方案特别适合：

1. 使用 Hugo 的个人博客用户

因为：Markdown → Hugo，天然兼容。

---

### 2. 经常移动写作的人

例如：通勤，上课，碎片记录

手机可以直接完成博客草稿。

---

### 3. 希望保留 Git 工作流的人

相比 Notion 类平台：
Git 方案：数据自主可迁移，可版本控制，可离线，长期维护性更强。

---

## 十、我的最终评价

如果目标是：
> “低成本、可长期维护、手机友好”的 Hugo 博客方案

那么：
Obsidian + GitSync + GitHub + Cloudflare Pages目前是我体验最均衡的一套方案。
它并不是最专业，最自动化，最强大。但它的优势在于足够简单
同时足够稳定，对于个人博客而言，这比“功能极致复杂”更重要。
