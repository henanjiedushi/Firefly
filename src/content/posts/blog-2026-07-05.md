## 🛠️ 核心准备工作

这套方案之所以能做到完全免费且自主掌控，是因为我们利用了 Git 的底层版本控制能力。在手机端正式开工前，需要准备好以下两样工具：

1. **GitHub 个人访问令牌（Personal Access Token, PAT）**
   由于 GitHub 移动端安全限制，手机插件无法通过传统的账号密码登录。
   * **生成路径**：GitHub 网页版 -> `Settings` -> `Developer settings` -> `Personal access tokens (classic)`。
   * **注意**：必须勾选 **`repo`** 权限。生成后复制那串以 `ghp_` 开头的 Token 备用（它只显示一次）。
2. **手机端 Obsidian 软件**
   在手机应用商店或官网下载安装好最新版的 Obsidian。

## 📦 步步为营：从零部署手机端 Git 环境

很多朋友听说手机端 Git 插件不好用，绝大多数是因为卡在了“第一次克隆仓库”的步骤上。手机端由于没有系统级 Git 环境，插件在底层使用的是纯 JavaScript 模拟网络请求。

以下是无痛克隆云端博客仓库的正确姿势：

### 1. 建立跳板空白库
在手机端打开 Obsidian，点击 **Create new vault（创建新仓库）**。起一个名字（如 `Blog-Sync`），选择一个本地存储路径，创建一个完全清空的库。

### 2. 安装并启用插件
进入库后，点击左下角齿轮进入设置 -> `Community plugins（社区插件）` -> 关闭安全模式 -> 点击 `Browse` 搜索 **`Obsidian Git`** 并下载启用。

### 3. 精准克隆云端仓库
回到 Obsidian 主界面，**在屏幕上向下滑动**呼出命令面板。输入 `Git Clone` 并执行 **`Obsidian Git: Clone an existing remote repository`**。
在顶部依次弹出的输入框中填写：
* **URL**：你的博客仓库 **HTTPS 格式** 链接（如 `https://github.com/用户名/仓库名.git`，*绝对不要填 SSH 链接*）。
* **Username**：你的 GitHub 用户名。
* **Password**：粘贴你刚刚准备好的 `ghp_` 开头的 Token。
* **Depth（深度）**：**强烈建议填写 `1`**（浅克隆）。这可以只下载最新的一层快照，避开几百兆的历史提交记录，极大防止手机因内存不足而闪退。

克隆完成后，界面会弹出绿色成功提示。此时**彻底杀掉手机 Obsidian 进程并重启**，你的所有历史博客文章就完美同步到手机上了！

## ✍️ 移动端日常写作与优雅配置

在手机上写博客，我们需要让 Markdown 文件的顶部元数据（Front Matter）符合 Hugo 等博客框架的解析标准。

### 1. 利用 Properties（属性）可视化填写
Obsidian 带来了优雅的 Properties 视图。你无需再手动枯燥地敲 YAML 代码，在文章第一行输入 `/` 选择 **添加文件属性（Add file property）**，或者呼出命令面板选择该功能。
建议固定以下几个博客高频属性：
* `title` (文本)：文章标题
* `date` (日期与时间)：博客发布时间（插件会自动帮你格式化为标准的 ISO 格式）
* `tags` (列表)：文章标签
* `draft` (布尔值)：是否为草稿

### 2. 完美的“半自动”日常同步工作流
为了省电以及避免在狭小的手机屏幕上痛苦地手动解决代码冲突，我不建议开启频繁的“后台自动同步”。我目前采用的**最佳实践工作流**如下：

* **开工前先拉取（Pull）**：今天准备写字时，打开手机 Obsidian 的第一件事，就是**向下滑动屏幕呼出命令面板**，输入 `Git Pull` 运行。确保把电脑端昨晚写的最新内容拉取到手机本地，做到“先拉后写，100% 避免冲突”。
* **完工后一键同步（Push）**：文章写完或修改完毕后，再次向下滑动屏幕，在命令面板中输入 `sync`，执行：
  > **`Obsidian Git: Commit and sync all changes`**
  
  这是一个插件封装好的“组合拳”。它会自动帮你的改动生成本地快照（Commit），并瞬间推送到 GitHub 云端（Push）。看到右上角绿色的 Success 弹窗，意味着你的博客已经开始自动构建并发布了。

## 💡 总结与避坑原则

通过这套方案，我成功实现了“口袋里的博客编辑器”。随时随地掏出手机，就能用最爱的双拼输入法在 Obsidian 里流畅敲字。

最后给想尝试这套方案的朋友总结 3 个“保命”原则：
1. **网络超时**：国内移动网络连接 GitHub 经常波动。如果同步时没有反应，记得去插件设置里把 **Network Timeout** 改为 `30000` 或 `60000`（延时 30-60 秒），并配合全局代理环境使用。
2. **多媒体附件**：由于手机端性能限制，尽量不要把数兆大小的高清原图或巨型 PDF 直接塞进库里让 Git 同步，推荐结合图床使用。
3. **保持纯净**：克隆前，手机端的文件夹必须是绝对干净的，否则会触发 `Directory not empty` 报错阻断克隆。
